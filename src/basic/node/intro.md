# node 简介

Node 是一个基于 Chrome V8 引擎的 JavaScript 运行时，它允许在服务器端运行 JavaScript 代码。它采用事件驱动和非阻塞 I/O 模型，使得构建高性能、可扩展的网络应用变得简单。

## 什么是 Node？

Node 不是一种新语言，而是一个运行环境。它将 JavaScript 从浏览器中解放出来，使其能够处理文件操作、网络请求等后端任务。Node 使用单线程事件循环机制，通过异步方式处理多个请求，避免阻塞。

## 关键概念

Node 的核心是事件驱动和非阻塞 I/O。事件循环不断检查事件队列，当有事件（如用户请求）到达时，它会触发回调函数进行处理，而不会等待 I/O 操作完成。

示意图：事件循环流程
```
请求 -> 事件队列 -> 事件循环 -> 执行回调 -> 返回响应
```

非阻塞 I/O 示意图：
```
客户端请求 -> Node 服务器 -> 异步处理 I/O（如读取文件） -> 立即返回控制权 -> 完成后通过回调发送响应
```

## 优点

- 高性能：事件驱动模型允许处理大量并发连接。
- 统一语言：使用 JavaScript 同时开发前端和后端，减少学习成本。
- 丰富生态：通过 npm（Node 包管理器）访问海量开源模块。

## 使用场景

Node 适用于实时应用，如聊天服务器、API 服务和数据流处理。它不适合 CPU 密集型任务，因为单线程可能阻塞。
